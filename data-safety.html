🛡️ KutumbTree Data Safety Statement

Last Updated: June 12, 2026

Data Safety Overview

At KutumbTree, protecting your family's data is our highest priority. This document outlines our data safety practices and security measures.

✓ HTTPS Encryption ✓ Password Hashing ✓ JWT Tokens ✓ Daily Backups

1. Data Collection

What We Collect

Data Type Collected Purpose Shared

Password ✓ Account authentication ✗
Family Member Info ✓ Family tree management Only with your consent
Photos/Media ✓ Display in family tree Only with your consent
Contact Info ✓ Invite family members Only with your consent

Usage Analytics ✓ Service improvement ✗
Location Data ✗ Not collected ✗

2. Encryption & Data Protection

In-Transit Security

  • HTTPS/TLS: All communication is encrypted using TLS 1.2 or higher
  • Certificate: Valid SSL/TLS certificate from trusted certificate authority
  • Handshake: Secure key exchange during connection establishment

At-Rest Security

  • Database: Data stored on secured database servers
  • Backups: Encrypted backup storage with restricted access
  • Access Control: Role-based access to database resources

Password Security

  • Algorithm: Bcrypt hashing with salt
  • Never Stored: Plain-text passwords never stored or logged
  • Minimum Length: 8 characters required
  • Complexity: Recommended mix of uppercase, lowercase, numbers, symbols

3. Authentication & Authorization

Login Security

  • JWT Tokens: JSON Web Tokens with 24-hour expiration
  • Refresh Tokens: Secure refresh token rotation for session extension
  • CORS Protection: Cross-Origin Resource Sharing properly configured

Access Control

  • User Isolation: Users can only access their own data
  • Family Sharing: Granular permissions for shared family trees
  • Admin Controls: Tools to manage access and revoke permissions

4. Data Retention & Deletion

Retention Policies

Data Type Retention Period Deletion Policy
User Accounts Until deletion Permanent on request
Family Tree Data Until deletion Permanent on request
Server Logs 30 days Auto-deleted after 30 days
Backups 90 days max Auto-purged after 90 days
Session Tokens 24 hours Auto-expired

Deletion Process

When you request account deletion:

  1. All personal data is permanently removed from active database within 24 hours
  2. Backups containing your data are purged within 90 days
  3. Server logs are automatically deleted after 30 days
  4. You receive confirmation email after deletion is complete

5. Data Sharing & Third Parties

We DO NOT Share Data With:

  • Advertisers or marketers
  • Data brokers
  • Social media platforms
  • Analytics services (for personal data)
  • Any commercial entities

Limited Sharing For Service Delivery:

  • Railway (Hosting): Servers host your data (encrypted)
  • Database Services: Data management infrastructure
  • Family Members: Only information you choose to share

Legal Compliance:

We may disclose data when legally required by:

  • Court orders or subpoenas
  • Law enforcement requests
  • Government agencies (with proper legal process)

6. Security Practices

Infrastructure Security

  • Firewall: Network firewall protection
  • DDoS Protection: Rate limiting and traffic monitoring
  • Access Logs: All access attempts logged and monitored
  • Intrusion Detection: Automated threat detection systems

Application Security

  • Input Validation: All user input validated and sanitized
  • SQL Injection Prevention: Prepared statements used
  • XSS Protection: Content Security Policy headers
  • CSRF Protection: Token-based protection

Monitoring & Updates

  • Security Patches: Applied within 24 hours of release
  • Dependency Updates: Regular updates of libraries and frameworks
  • Security Scanning: Regular vulnerability scans
  • 24/7 Monitoring: Continuous monitoring for threats

  2. Investigate the scope and cause of the breach
  3. Notify affected users within 72 hours
  4. Provide guidance on protective measures
  5. Cooperate with law enforcement if required

If you discover a security vulnerability, please responsibly disclose it to:
