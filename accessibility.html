♿ KutumbTree Accessibility Statement

Last Updated: June 12, 2026

Our Commitment to Accessibility

KutumbTree is committed to making our family tree management application accessible to all users, including those with disabilities. We believe everyone should have equal access to preserve and share their family history, regardless of their abilities.

This Accessibility Statement describes the accessibility features and practices of KutumbTree. We continuously work to improve accessibility and welcome feedback from our users.

1. Accessibility Features

Vision & Sight

  • Dark Mode: Reduces eye strain with light text on dark background
  • High Contrast Mode: Enhanced colors for better visibility
  • Adjustable Text Size: Pinch-to-zoom works throughout the app
  • Screen Reader Compatible: Full compatibility with VoiceOver (iOS) and TalkBack (Android)
  • Image Descriptions: All family tree images have alt text descriptions
  • Keyboard Navigation: Complete keyboard-only navigation support
  • Focus Indicators: Clear visual indication of focused elements
  • Color Not Alone: Information is not conveyed by color alone

Hearing & Auditory

  • No Sound Required: All features work without audio
  • Transcripts: Any instructional videos include captions and transcripts
  • Visual Feedback: Notifications use visual indicators, not just sound

Motor & Mobility

  • Touch-Friendly: Large touch targets (minimum 48×48 pixels)
  • Keyboard Support: Full keyboard navigation without mouse required
  • Gesture Alternatives: All gestures have non-gesture alternatives
  • Voice Input: Compatible with device voice input methods
  • Switch Access: Compatible with switch control devices (iOS/Android)

Cognitive & Neurological

  • Simple Navigation: Clear, intuitive menu structure
  • Consistent Design: Predictable button and control placement
  • Clear Language: Plain language in instructions and messages
  • Undo Functionality: Ability to undo most actions
  • Help & Support: Easy access to contextual help

2. Compliance Standards

We Aim to Meet:

  • WCAG 2.1 Level AA: Web Content Accessibility Guidelines (our target)
  • Section 508: U.S. Rehabilitation Act requirements
  • EN 301 549: European accessibility standard
  • Android Accessibility Guidelines: For Android app version
  • iOS Accessibility Guidelines: For iOS app version
Note: While we strive to meet WCAG 2.1 AA standards, some features may still need improvement. We actively work to remediate accessibility issues as they are discovered.

3. Accessibility Features by Device

iOS App

  • VoiceOver: Full support for Apple's built-in screen reader
  • Zoom: Magnification up to 15x
  • Display & Text Size: Dynamic Type support
  • Color Invert: Invert colors option for light sensitivity
  • Reduce Motion: Respects motion reduction preference
  • Hearing Aid Compatibility: Works with hearing aids

Android App

  • TalkBack: Full support for Google's screen reader
  • Select to Speak: Tap text to hear it read aloud
  • Magnification: Magnification gesture support
  • High Contrast Text: Bold and high contrast options
  • Font Size: System font size adjustments respected
  • Caption Preferences: System caption preferences honored

Web App

  • Screen Readers: NVDA, JAWS, and VoiceOver supported
  • Keyboard Navigation: Complete with Tab order
  • Browser Zoom: Works with browser zoom up to 200%
  • High Contrast Mode: Windows High Contrast mode supported

4. Known Accessibility Issues

We are aware of the following areas that may need improvement:

  • Tree visualization canvas: Limited screen reader support (we're working on this)
  • Some custom date inputs: May not work with all screen readers
  • Drag-and-drop features: Not accessible via keyboard (alternative methods provided)

We are actively working to address these issues. If you discover others, please report them (see Contact Us).

5. Browser & Device Compatibility

Supported Devices

  • iOS: Version 14.0+
  • Android: Version 8.0+ (API 26+)
  • Web: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+

Assistive Technology

  • VoiceOver (iOS 14+)
  • TalkBack (Android 8+)

  • NVDA (Windows)
  • VoiceOver (macOS)
  • Switch Control (iOS/Android)

10. Report Accessibility Issues

Found Something Not Accessible?

Please report it to:

Email: accessibility@kutumbtree.com

Include:

  • Device and OS version
  • App version (or web browser)
  • Specific feature or page with issue
  • Assistive technology used (if applicable)
  • How you expected it to work

Response Time: We aim to acknowledge accessibility reports within 24 hours and provide updates every week
